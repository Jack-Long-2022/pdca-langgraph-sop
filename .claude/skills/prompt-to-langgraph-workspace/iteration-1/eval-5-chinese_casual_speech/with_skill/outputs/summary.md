# 中文随意语言处理总结

## 任务概述
将用户使用中文随意口语描述的"自动写报告流程"转换为结构化的 LangGraph 工作流配置。

## 用户输入分析

### 原始输入
> "那个，我想做一个自动写报告的流程。首先呢，用户输入一个主题，然后系统去搜索相关的资料，搜完之后呢，把结果整理一下，然后让AI分析生成一个报告大纲。大纲生成之后，如果用户觉得不满意，可以让AI重新生成，最多重试3次。大纲确定后，再根据大纲写完整的报告内容。"

### 输入特征
1. **高度口语化**：包含大量填充词("那个"、"首先呢"、"然后呢")
2. **逻辑清晰**：虽然口语化，但步骤顺序明确
3. **包含条件分支**：用户确认+重试机制
4. **包含循环控制**：最多重试3次的明确限制

## 预处理结果

### 清洗操作
- **移除填充词**："那个"、"首先呢"、"然后呢"、"搜完之后呢"
- **标准化连接词**："然后" → "完成后"
- **提取关键动作**：6个核心步骤

### 关键动作识别
1. 用户输入一个主题
2. 系统去搜索相关的资料
3. 把结果整理一下
4. AI分析生成一个报告大纲
5. 用户确认大纲(不满意可重试最多3次)
6. 根据大纲生成完整报告

## 节点识别结果

### 显式节点 (6个)
1. **接收主题输入** (control_node/input_node) - 置信度: 0.98
2. **搜索相关资料** (tool_node/search) - 置信度: 0.99
3. **整理搜索结果** (thinking_node/extraction) - 置信度: 0.95
4. **生成报告大纲** (thinking_node/generation) - 置信度: 0.99
5. **用户审核大纲** (control_node/human_approval) - 置信度: 0.97
6. **生成完整报告** (thinking_node/generation) - 置信度: 0.99

### 隐式节点 (2个)
1. **重试控制** (control_node/loop_node) - 置信度: 0.92
   - 推断依据：提到"最多重试3次"
2. **输出报告** (control_node/output_node) - 置信度: 0.90
   - 推断依据：工作流需要明确的出口

## 边关系识别

### 边类型统计
- **顺序边 (sequential)**: 5条
- **条件边 (conditional)**: 1条
- **循环边 (loop)**: 1条

### 关键条件识别
- `{user_approval} == true` → 继续生成报告
- `{user_approval} == false` → 进入重试循环
- `{retry_count} >= 3` → 终止重试

## 状态定义

### 状态分类
- **输入状态 (1个)**: topic - 用户输入的主题
- **中间状态 (4个)**: search_results, organized_data, outline, user_approval
- **控制状态 (2个)**: retry_count, workflow_id, timestamp
- **输出状态 (1个)**: final_report - 最终报告

## 推断和假设

### 主要推断
1. **循环控制节点**: 管理3次重试限制
2. **输出节点**: 完成工作流的最终输出
3. **条件分支补全**: 用户满意/不满意的双向分支

### 关键假设
1. 使用综合网络搜索作为资料来源
2. 整理包括去重、分类和提取关键信息
3. 用户审核采用明确批准/拒绝机制
4. 报告输出为Markdown格式
5. 重试基于用户反馈进行优化

## 歧义点标注

### 识别的歧义 (4个)
1. **搜索来源**: 未明确使用哪个搜索引擎
   - 建议: 网络搜索/内部知识库/学术数据库
2. **整理标准**: "整理一下"的具体含义
   - 建议: 去重和分类/提取关键信息/生成摘要
3. **用户交互**: 用户如何表达不满意
   - 建议: 明确拒绝/评分系统/提供修改建议
4. **报告格式**: 最终报告的格式要求
   - 建议: Markdown/PDF/纯文本/HTML
