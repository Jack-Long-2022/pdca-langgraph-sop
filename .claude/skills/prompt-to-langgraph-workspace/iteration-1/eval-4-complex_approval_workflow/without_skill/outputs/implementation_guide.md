# 复杂审批工作流 - 实现指南

## 工作流概述

本工作流实现了一个完整的审批流程，从用户申请到管理员审核再到邮件通知。

### 核心流程步骤

1. **用户提交申请** - 用户填写申请表单
2. **系统验证** - 验证用户权限、数据完整性和业务规则
3. **条件分支** - 根据验证结果决定下一步
4. **管理员审核** - 人工审核决策（批准/拒绝/请求信息）
5. **邮件通知** - 发送审核结果通知
6. **状态更新** - 更新申请的最终状态

## 工作流特点

### 🔄 条件分支
- **验证检查点**: 系统验证后，根据结果决定是发送给管理员还是返回错误
- **管理员决策点**: 管理员可以选择批准、拒绝或请求更多信息

### 👥 人工交互
- **管理员审核节点**: 需要人工介入进行审核决策
- **支持委托**: 管理员可以将审核任务委托给其他人
- **超时升级**: 超过规定时间自动升级给高级管理员

### 📧 通知系统
- **多渠道通知**: 支持邮件、仪表板、短信等多种通知方式
- **优先级路由**: 紧急申请自动路由给高级管理员
- **错误重试**: 邮件发送失败时自动重试

### ⚡ 错误处理
- **验证错误**: 返回详细的错误信息和修正建议
- **系统错误**: 记录错误并联系技术支持
- **通知错误**: 使用备用通知方式继续流程

## 状态管理

### 初始状态
```json
{
  "application": null,
  "validation_passed": false,
  "admin_decision": null,
  "final_status": "pending"
}
```

### 最终状态
- `approved` - 申请已批准
- `rejected` - 申请已拒绝
- `error` - 处理过程中发生错误

### 中间状态
- `pending_validation` - 等待系统验证
- `pending_approval` - 等待管理员审核
- `info_requested` - 需要用户提供更多信息

## 快乐路径流程

```
开始 → 用户提交申请 → 系统验证 → 验证通过 → 发送给管理员
→ 管理员批准 → 发送批准邮件 → 更新状态 → 结束
```

## 异常处理路径

### 1. 验证失败
```
系统验证 → 验证失败 → 返回错误信息 → 结束
```

### 2. 管理员拒绝
```
管理员审核 → 拒绝 → 通知拒绝原因 → 结束
```

### 3. 需要更多信息
```
管理员审核 → 请求信息 → 通知用户 → 返回申请提交 → 重新提交
```

### 4. 邮件发送失败
```
发送批准邮件 → 邮件失败 → 重试或备用通知 → 继续更新状态
```

## 配置参数

### 验证配置
- `strict_mode`: 严格模式开关
- `validation_checks`: 验证检查项列表
- `allow_correction`: 是否允许用户修正后重新提交

### 审核配置
- `admin_roles`: 允许审核的管理员角色
- `approval_deadline`: 审核截止时间（小时）
- `require_comment_on_reject`: 拒绝时是否要求填写原因

### 通知配置
- `channels`: 通知渠道列表
- `templates`: 邮件模板配置
- `retry_config`: 重试策略配置

### 邮件配置
- `smtp_server`: SMTP服务器地址
- `from_address`: 发件人地址
- `templates_path`: 模板文件路径

## 集成点

### 1. 邮件服务 (SMTP)
- **目的**: 发送批准通知和错误提醒
- **协议**: SMTP
- **功能**: 批准通知、拒绝通知、信息请求

### 2. 用户管理系统 (REST API)
- **目的**: 验证用户权限和获取管理员信息
- **协议**: REST API
- **功能**: 权限检查、用户信息查询

### 3. 仪表板系统 (WebSocket)
- **目的**: 显示待审核任务和状态更新
- **协议**: WebSocket
- **功能**: 实时推送审核任务、状态更新

## 性能考虑

### 预期处理时间
- **最快路径**: 1-2分钟（验证失败）
- **正常路径**: 1-3天（包含人工审核）
- **最慢路径**: 7天以上（包含超时升级）

### 并发处理
- 支持多个申请同时处理
- 每个申请独立状态管理
- 管理员可以并行审核多个申请

## 错误代码

| 错误代码 | 描述 | 处理方式 |
|---------|------|---------|
| VAL_001 | 用户权限不足 | 返回权限错误，建议联系管理员 |
| VAL_002 | 数据不完整 | 返回缺失字段列表，允许重新提交 |
| VAL_003 | 业务规则违规 | 返回违规详情，提供修正建议 |
| SYS_001 | 系统内部错误 | 记录错误日志，联系技术支持 |
| NET_001 | 网络连接失败 | 重试操作，使用备用通知方式 |
| EMAIL_001 | 邮件发送失败 | 重试发送，降级到仪表板通知 |

## 监控指标

### 关键指标
1. **平均处理时间**: 从提交到最终决策的时间
2. **审核通过率**: 批准申请占审核申请的比例
3. **系统错误率**: 系统错误占总流程的比例
4. **邮件发送成功率**: 邮件发送成功的比例

### 告警规则
- 处理时间超过7天
- 系统错误率超过5%
- 邮件发送失败率超过10%
- 待审核任务超过100个

## 安全考虑

1. **数据加密**: 传输和存储敏感数据时使用加密
2. **访问控制**: 严格的角色权限管理
3. **审计日志**: 记录所有关键操作
4. **数据脱敏**: 日志中敏感信息脱敏处理
